# Config Directory

Management of configuration files for the local-network Docker Compose stack.

## Structure

```
config/
├── shared/              # Shared utilities (committed to git)
│   └── lib.sh           # Shell functions (require_jq, etc.)
└── README.md
```

Generated contract address files (horizon.json, subgraph-service.json, etc.) are stored
in the `config-local` Docker volume, not on the host filesystem. This ensures
`docker compose down -v` produces a fully clean state.

## How It Works

### Mount Layout

Services receive two directory mounts plus a direct `.env` mount:

| Source            | Container path     | Mode                                | Contents                                           |
| ----------------- | ------------------ | ----------------------------------- | -------------------------------------------------- |
| `config/shared/`  | `/opt/shared/`     | `:ro`                               | Git-tracked utilities (`lib.sh`)                   |
| `.env`            | `/opt/config/.env` | `:ro`                               | Environment variables (ports, mnemonics, versions) |
| `config-local` volume | `/opt/config/` | `:ro` (`:rw` for `graph-contracts`) | Generated contract address files                   |

Scripts source the env file and shared library:

```sh
. /opt/config/.env
. /opt/shared/lib.sh
token=$(require_jq '."1337".L2GraphToken.address' /opt/config/horizon.json)
```

Note: `env_file` cannot be used because several binaries (graph-node, indexer-service-rs)
interpret environment variables as configuration flags, causing collisions with `.env` entries.

### Contract Deployment (`graph-contracts` service)

- Creates empty config files on first run (if they don't exist)
- Deploys Graph protocol smart contracts (Horizon) using Hardhat Ignition
- Deploys TAP contracts (Escrow, TAPVerifier, AllocationIDTracker) using Forge
- Deploys DataEdge and REO contracts
- Writes contract addresses to the `config-local` volume:
  `horizon.json`, `subgraph-service.json`, `issuance.json`, `tap-contracts.json`, `block-oracle.json`

### Inspecting Config from the Host

Since config files live in a Docker volume, use `docker exec` to inspect them:

```bash
docker exec graph-node cat /opt/config/horizon.json | jq .
```

## Files

### `shared/lib.sh`

- **Committed to git**
- **Contains:** Shared shell functions (`require_jq`)
- **Mounted in:** Services as `/opt/shared/lib.sh:ro`

### Address Books (on `config-local` volume)

| File                     | Generated by      | Contents                                              |
| ------------------------ | ----------------- | ----------------------------------------------------- |
| `horizon.json`           | `graph-contracts` | Horizon protocol contract addresses                   |
| `subgraph-service.json`  | `graph-contracts` | Subgraph service contract addresses                   |
| `tap-contracts.json`     | `graph-contracts` | TAP contract addresses (Escrow, TAPVerifier, etc.)    |
| `block-oracle.json`      | `graph-contracts` | DataEdge contract address                             |
| `issuance.json`          | `graph-contracts` | REO contract address                                  |

## Cleaning

```bash
docker compose down -v
```

This removes all Docker volumes (chain data, postgres, config-local, etc.),
giving a fully clean state for `docker compose up`.
